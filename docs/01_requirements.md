<a id="purpose"></a>
## 1. 目的

AlphaLens JPは、日本株の企業調査を効率化するAIファンダメンタルズ分析SaaSです。

ユーザーは銘柄を検索し、財務データ・株価データ・企業情報を確認し、AIに調査レポートを生成させることができます。分析結果は保存でき、後からWatchlistや分析履歴として見返せます。

<a id="background"></a>
## 2. 背景

日本株の調査では、銘柄情報、株価、財務、決算発表、開示資料、ニュースなどの情報が分散しています。個人投資家やリサーチ担当者は、複数サイトを横断しながら情報を集め、手作業で要約・比較する必要があります。

AlphaLens JPは、情報収集と初期分析をSaaS化し、調査の初動を速くします。

<a id="users"></a>
## 3. ターゲットユーザー

- 日本株を調査する個人投資家
- 企業分析の学習者
- ファンダメンタルズ分析を効率化したいユーザー
- 決算や財務指標を継続的に確認したいユーザー

採用向けの見せ方では、金融ドメイン特化の個人ツールではなく、データ統合・AI要約・ダッシュボードSaaSとして説明します。

<a id="mvp-scope"></a>
## 4. MVPスコープ

MVPで実装する機能は次です。

| 区分 | 機能 | 概要 |
| --- | --- | --- |
| 認証 | ログイン/ログアウト | Node.js / TypeScript APIでメール/パスワード認証を行い、HttpOnlyセッションCookieでユーザーを識別する |
| 銘柄検索 | 日本株検索 | 銘柄コード、企業名、業種で検索する。4桁入力とJ-Quantsの5桁銘柄コードの正規化に対応する |
| 銘柄詳細 | 企業基本情報 | 企業名、市場、業種、銘柄コードを表示する |
| 株価 | 日足チャート | 調整済み終値、出来高を表示する |
| 財務 | 財務サマリ | 売上、営業利益、純利益、EPS、自己資本比率などを表示する |
| 分析 | AIレポート生成 | 財務・株価データをもとに調査メモを生成する |
| 保存 | 分析履歴 | 生成したAIレポートを保存する |
| 保存 | Watchlist | 気になる銘柄を保存する |
| 運用 | API取得ログ | 外部API取得結果、失敗、再試行を記録する |

<a id="non-mvp"></a>
## 5. 非MVPスコープ

MVPでは次を実装しません。

- 投資助言
- 買い/売りの推奨
- 株価予測
- 証券口座連携
- 自動売買
- リアルタイム株価
- 有料課金
- 複数人チーム管理
- Nansenなど暗号資産オンチェーン分析
- 決算短信PDFの全文RAG

<a id="functional-requirements"></a>
## 7. 機能要件

### 7.1 認証

- ユーザーはログインできる。
- ログインしていないユーザーはWatchlistと分析履歴を保存できない。
- APIは認証が必要なエンドポイントを保護する。
- ユーザーごとのデータは分離する。
- 認証はNode.js / TypeScript APIが発行する不透明なセッショントークンをHttpOnly Cookieで保持する。
- `POST`、`PUT`、`PATCH`、`DELETE` にはDouble Submit Cookie方式でCSRF対策を行う。
- 公開デモでは登録APIのレート制限を行う。必要に応じて `REGISTRATION_ENABLED=false` で新規登録を停止し、デモユーザーを用意する。

### 7.2 銘柄検索

- 銘柄コード、企業名、英語企業名、業種で検索できる。
- 銘柄コードはユーザー向け表示コードとデータプロバイダ向けコードを分けて扱う。
- 検索結果は最大50件を返す。
- 上場廃止銘柄はMVPでは対象外としてよい。

### 7.3 銘柄詳細

- 企業基本情報を表示する。
- 株価チャートを表示する。
- 財務サマリを表示する。
- AI分析レポート生成ボタンを表示する。

### 7.4 財務分析

- 直近3年または取得可能な期間の財務データを表示する。
- 売上高、営業利益、経常利益、純利益、EPS、BPS、自己資本比率などを表示する。
- 取得できない指標は空欄または「データなし」と表示する。
- 派生指標はバックエンドで計算し、計算式をコードに残す。

### 7.5 AIレポート

- AIは、成長性、収益性、安全性、リスク、確認すべき点を出力する。
- AIは、買い/売り推奨や価格予測を出力しない。
- AIレポートには、利用したデータの取得日時を表示する。
- AIレポートには、根拠にした主要指標と対象期間を表示する。
- AIレポートはユーザー単位で保存する。

### 7.6 Watchlist

- ログインユーザーは銘柄をWatchlistに追加できる。
- Watchlistから銘柄を削除できる。
- Watchlistには直近の株価、前日比、最終分析日時を表示する。

<a id="non-functional-requirements"></a>
## 8. 非機能要件

| 区分 | 要件 |
| --- | --- |
| パフォーマンス | 主要画面は通常時2秒以内に初期表示する |
| 可用性 | MVPでは個人利用想定。単一リージョンでよい |
| セキュリティ | APIキー、OpenAI APIキー、DBパスワードはSecrets Managerまたは環境変数で管理する |
| 保守性 | 外部APIクライアントはインターフェース化し、J-Quants以外に差し替え可能にする |
| 監視 | APIエラー、AI生成失敗、外部API制限超過をログに残す |
| コスト | 個人開発のため、月額数千円から1万円程度に抑える設計を優先する |
| 法務/表現 | 投資助言ではない旨をUIとREADMEに明記する |
| データ鮮度 | MVPでは日次データを扱い、リアルタイム株価として見せない |

<a id="data-source-requirements"></a>
## 9. データソース要件

主データソースはJ-Quants APIです。

J-Quants APIから取得する候補データは次です。

- 上場銘柄情報
- 株価四本値
- 財務情報
- 決算発表予定日

J-Quants APIはV2移行が進んでいるため、実装前に現行仕様を確認してください。MVPでは、APIクライアント層を `MarketDataProvider` として抽象化し、V2/APIキー方式を既定にします。旧V1仕様は明示的な互換モードに留め、EDINETや他のAPIにも差し替えられるようにします。

J-Quants連携で注意する点:

- APIで扱う銘柄コードは4桁入力と5桁コードが混在しうるため、内部では `provider_code` と `display_code` を分ける。
- 株価四本値は日次データであり、OHLCや出来高がNullになる日がある。
- 取得可能な期間、財務項目、利用量は契約プランに依存するため、MVPはMock Providerでも動くようにする。
- J-Quants実連携はAIレポート生成より前に実装し、AIが実データとMockデータを区別できるようにする。

<a id="ai-requirements"></a>
## 10. AI要件

- 入力は、構造化した財務データ、株価サマリ、企業基本情報に限定する。
- AI出力は、OpenAI Responses APIで生成する。
- AI出力は、Structured OutputsでJSON Schemaに沿った調査メモ形式とする。
- AI出力には免責文を含める。
- AIが存在しないデータを断定しないように、欠損データは明示的に渡す。
- AI生成結果はDBに保存し、同一データで再生成しすぎないようにする。

<a id="constraints"></a>
## 11. 制約・注意事項

- 本サービスは投資判断を代替しない。
- 金融商品取引法上の投資助言に該当しないよう、売買推奨・目標株価・将来収益の断定を避ける。
- 外部APIの利用規約、レート制限、料金プランを遵守する。
- J-Quantsの無料枠では取得期間や利用可能データが制限される可能性がある。
- MVPでは外部APIの生レスポンスをDBに保存しない。保存する場合は、利用規約上の保存可否を確認した後の将来拡張とする。
- AIの出力は誤る可能性があるため、根拠データと取得日時を併記する。

<a id="success-criteria"></a>
## 12. 成功基準

MVP完成の成功基準は次です。

- 主要な日本株銘柄を検索できる。
- 企業詳細画面で株価・財務・AI分析を確認できる。
- J-Quants実データまたは明示されたMockデータで、銘柄詳細からAI分析まで一貫して動く。
- AIレポートが投資助言ではなく調査メモとして生成される。
- Watchlistと分析履歴がユーザー別に保存される。
- AWS上の公開URLでデモできる。
- READMEに技術選定、構成図、API設計、DB設計、テスト結果が書かれている。
